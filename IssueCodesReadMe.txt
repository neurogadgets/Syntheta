These codes are used within comments in the code, to indicate to-dos:

## = for later consideration
### = non-blocking to-do
###! = needs attention (priority to-do)
###!! = known bug (priority blocking to-do)

###? = needs R&D

###T = needs testing
###TL = logic needs testing
###TP = policy to revisit

###O = may be optimized

###D = debug code
